The course

One primitive — a board and its moves — and a chapter of Terraform per lesson. Each chapter is a directory in examples/ with a README of missions and hints. You write every line of Terraform yourself; keep this console visible while you do. (Stuck? An answer key lives on the solutions branch.)

1Terraform basics

examples/chapter-1-basics

Resource blocks, references between resources, variables, outputs, and the plan → apply → destroy loop. Your tools: the fakecloud_tictactoe_board resource (just needs a name) and fakecloud_tictactoe_move (board_id, player, position 0-8). Draw your first marks and watch each command change real state here.

2count, for_each & the footguns

examples/chapter-2-count-vs-foreach

Draw a whole line of marks from one block with count — then remove the first element of the list and read the plan very carefully before you apply it. The board makes the index-shift footgun visible. Then rebuild the block with for_each so each mark's identity is its cell, not its list position, and run the same experiment.

3Drift & import

examples/chapter-3-drift-and-import

This console is your coworker who clicks things. Click an empty cell on a freeplay board to make a mark Terraform doesn't know about (unmanaged infra — note that terraform plan says nothing!). Click one of your managed marks to delete it behind Terraform's back — now plan detects drift and offers to replay it. Finally, adopt a clicked mark with terraform import — every mark below lists its id.

4Modules

examples/chapter-4-modules

Write a glyph module — variables in, board + marks inside, outputs back — then stamp patterns from single module blocks. for_each over a map of glyphs conjures a whole gallery in one apply; watch this console when you run it. That's the moment modules click.

5Shared state & stepping on each other

examples/chapter-5-shared-state

The 🪧 nameplate is the only fakecloud resource that updates in place. Two players import the same plaque into two state files and start an apply war — watch the yellow ~ lines ping-pong in the feed. Then the fixes: ignore_changes, single ownership + data sources, and why real teams use state locking. Plus prevent_destroy and the create_before_destroy footgun.

6Dependencies & ordering

examples/chapter-6-dependencies

Terraform runs a graph, not a script. References are the edges — watch this feed during apply: boards always land before their marks, independent things race in parallel. depends_on adds edges by hand, and a data source referencing your opponent's board (data.fakecloud_tictactoe_board.opponent.id) shows how the graph reaches infrastructure you don't manage — and where it stops.

★Side quest: the duel

examples/side-quest-duel

A board with mode = "duel" is refereed: X starts, turns alternate, the board locks on a win. Share the board id with a friend pointing at the same fakecloud and play one terraform apply at a time. Applying out of turn fails with the reason — a failed apply just means "wait for your opponent".
