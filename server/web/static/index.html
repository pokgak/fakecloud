Learn Terraform here

1Point Terraform at fakecloud

Build the provider from provider/ and add a dev override so Terraform uses your local build (see the repo README), then configure it:

provider "fakecloud" {
  endpoint = "http://localhost:8000"
}
2Create your first resource

terraform apply this and watch a card appear on the left — that's Terraform calling the fakecloud API:

resource "fakecloud_vm" "web" {
  name          = "web-1"
  instance_type = "t2.micro"
}
3Change it, destroy it

Edit instance_type and apply again — the feed shows a yellow ~ change. Then terraform destroy and watch the card disappear (red −).

4Cause some drift

Use the + create out-of-band or delete buttons in this console, then run terraform plan. Terraform notices the API state no longer matches its state file — that's drift, and the plan proposes fixing it.

5Side quest: tic-tac-toe over Terraform

Moves are resources. Create a game, share its id with a friend pointing at the same fakecloud, and take turns applying moves — the board updates live here. X goes first; the server rejects out-of-turn applies.

resource "fakecloud_tictactoe_move" "m1" {
  game_id  = 1
  player   = "X"
  position = 4  # center; cells are 0-8, row by row
}

terraform destroy a move to take it back. Deleting the game resets the board for everyone.
